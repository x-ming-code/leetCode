# 每日算法练习 | Daily Algorithm Practice

欢迎来到我的算法练习仓库！这里记录了我每天刷算法题的Java代码和解题思路，主要用于保持编程手感、积累解题经验，同时也希望能为同样在学习算法的同学提供一些参考。


## 📚 仓库说明

- **目的**：坚持每日一道（或多道）算法题，系统提升逻辑思维和代码实现能力，覆盖常见数据结构（数组、字符串、哈希表、链表、树等）和算法思想（双指针、贪心、动态规划、滑动窗口等）。
- **题目来源**：主要来自LeetCode经典题目，包含不同难度（简单至中等为主），每题代码中会注明题目描述和核心需求。
- **编程语言**：以Java为主，代码风格遵循Java编码规范，包含清晰的思路注释、测试用例和关键步骤说明，方便理解和复用。


## 📂 仓库结构

```
.
└── leetCode/src/main/java/com/ming/leetcode
    ├── code/seven/day22  # 按年月日期划分的包路径（示例：7月22日）
    │   └── Solution.java  # 当日算法题解（含题目描述、思路、代码）
    ├── code/ten/day9     # 10月9日的题目
    │   └── ThreeSum.java  # 三数之和题解
    └── ...  # 其他日期的题解
```

- 按**日期（年月）** 划分Java包路径，清晰追踪每日学习轨迹；
- 每个题目代码文件中包含：题目描述、核心解题思路、完整实现代码及测试用例；
- 思路说明涵盖：问题分析、算法选择理由、边界情况处理、时间/空间复杂度分析。


## 💡 解题原则

1. **先理后码**：先明确题目输入输出、约束条件和核心需求，再设计解题思路，避免盲目编码；
2. **多解对比**：同一题目尝试不同解法（如暴力解与优化解），通过注释对比优劣（如双指针vs哈希表）；
3. **可读性优先**：变量命名清晰（如left/right指针、currentSum累计和），关键步骤添加注释，便于复盘；
4. **定期复盘**：每周整理同类题目解题模板（如滑动窗口通用框架、双指针应用场景），提炼共性规律。
